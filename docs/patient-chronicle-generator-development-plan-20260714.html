01. 실행 요약

핵심 제품은 “문서 생성기”가 아니다. 환자에게 일어난 사건, 관찰된 내용, 당시 알려진 지식, 수행된 처치와 그 근거를 시간순으로 보존하는 Patient Chronicle Engine이다. 초진기록지·구급일지·NEDIS는 이 원장에서 서로 다른 지식 범위와 형식 규칙으로 생성되는 투영물이다.

최우선 결정: LLM이 환자와 문서를 한 번에 쓰게 하지 않는다. 코호트 분포 엔진이 환자 설계도를 만들고, 임상 전방 시뮬레이터가 Chronicle을 만들며, LLM은 제한된 구조화 확장과 문장화만 담당한다.

개별 사실성

한 환자 내부의 시간, 인과, 측정, 처치, 진단 변화와 문서 간 일관성을 보장한다.

코호트 사실성

100명 모두가 극적이지 않도록 주변·결합·조건부·시간·결측 분포를 별도로 통제한다.

감사 가능성

모든 값과 문장은 출처, 생성 규칙, 모델, 시드, 수정 이력을 역추적할 수 있어야 한다.

권장 구현 형태

  • Canonical store: PostgreSQL append-only 이벤트·Claim 저장소.
  • Batch analytics: DuckDB 기반 분포 분석, 코호트 감사, NEDIS 비교.
  • Graph: 별도 그래프 DB보다 먼저 typed edge 테이블과 materialized graph view 도입.
  • Rules: 시간·지식 경계·임상 불변식·NEDIS 조건부 필드를 선언형 카탈로그로 관리.
  • LLM: 구조화 JSON 생성, 제한된 dependency scope, 검증 실패 시 국소 repair.
  • Documents: Chronicle에서 파생. 생성 문서를 다시 임상 사실의 원본으로 사용하지 않음.

03. 제품 범위와 성공 기준

제품 목표

  • 한 환자의 baseline부터 여러 care episode와 encounter를 연결한 longitudinal Chronicle 생성.
  • 119 현장·이송, 응급실 초진, 검사·처치·진단 변화, 퇴실·입원 결과를 전방 시뮬레이션.
  • 동일 Chronicle에서 구급활동일지, 초진기록지, NEDIS 4.0 호환 레코드 생성.
  • 환자 1명의 임상 정합성과 100명 이상의 코호트 분포를 동시에 평가.
  • 변수·관계·기준값·코드·규칙을 버전 관리하고 재현 가능한 generation run 제공.

명시적 비목표

  • 실제 환자의 진단·치료를 자동 결정하는 임상 의사결정 시스템.
  • 의료인의 검토·서명 없이 실제 진료기록을 자동 작성·저장하는 기능.
  • 합성 데이터만으로 임상 연구의 유효성을 주장하는 기능.
  • 첫 버전부터 모든 질환·모든 연령·모든 병원 워크플로를 포괄하는 것.
  • 그래프 DB, 벡터 DB, 에이전트 프레임워크 자체를 제품 목표로 삼는 것.

제품 수준 성공 기준

  • 동일 seed와 동일 버전에서 구조화 Chronicle이 재현된다.
  • 필수 clinical·temporal·knowledge invariant의 critical violation이 0건이다.
  • 문서에 미래 정보가 유출되지 않는다.
  • 코호트의 주요 주변·조건부·시간·결측 분포가 정의된 허용 구간을 통과한다.
  • 각 문서 필드가 하나 이상의 Claim·Observation·Unknown 또는 명시적 projection rule로 추적된다.
  • 과표집 데이터는 selection probability와 sample weight를 제공한다.

04. 설계 원칙

Chronicle first

문서는 결과물이다. 문서끼리 서로 맞추지 않고 공통 원장에 각각 맞춘다.

Claim, not truth

환자 진술·보호자 진술·기기 측정·의사 판단을 덮어쓰지 않고 출처별 Claim으로 보존한다.

Forward-only clinical simulation

잠재 상태에서 관찰과 행동이 시간순으로 발생한다. 결과를 정한 뒤 과거를 교과서적으로 꾸미지 않는다.

Unknown is data

공란을 null로 축약하지 않는다. 질문 안 함, 측정 실패, 당시 미상, 충돌, 비대상 등을 구분한다.

Cohort before prose

전체 환자의 Blueprint와 분포를 먼저 승인한 뒤 비싼 상세·문장 생성을 수행한다.

Rules own safety

시간, 코드, 범위, 지식 경계, 조건부 필드는 결정적 규칙이 판정한다. LLM은 최종 권위가 아니다.

Append, supersede, never erase

수정은 원본 삭제가 아니라 새로운 버전과 supersedes 관계로 남긴다.

Version everything

스키마, 변수, 코드 체계, 기준값, 분포, 프롬프트, 모델, seed, adapter를 generation run에 고정한다.

05. 온톨로지와 토폴로지

12.1 Cohort-first sampler

function generate_cohort(contract):
    manifest = freeze_versions_and_seed(contract)
    world = simulate_world_timeline(contract.setting, contract.period)
    targets = load_distribution_targets(contract.mode, contract.population)

    blueprints = joint_sample(
        n = contract.population_size,
        conditional_dag = targets.conditional_dag,
        shared_context = world.context_events,
        soft_constraints = targets.tolerances,
        seed = manifest.seed.child("blueprints")
    )

    blueprint_audit = audit_blueprints(blueprints, targets)
    blueprints = resample_minimal_set(blueprints, blueprint_audit)

    chronicles = parallel_map(forward_simulate, blueprints)
    documents = project_all(chronicles, contract.document_profiles)
    final_audit = audit_all(chronicles, documents, targets)

    while final_audit.has_repairable_critical_findings:
        plan = choose_min_cost_repair(final_audit)
        apply_dependency_scoped_regeneration(plan)
        final_audit = audit_impacted_scope(plan)

    return release_if_gates_pass(manifest, chronicles, documents, final_audit)

표집 순서

  1. 기관·기간·계절·시간대·용량을 포함한 world state를 먼저 생성한다.
  2. 연령·성별·지역 같은 demographic strata를 표집한다.
  3. baseline condition과 persona를 조건부로 표집한다.
  4. 급성 episode family와 latent severity를 표집한다.
  5. arrival mode, KTAS, intervention density, disposition을 joint model로 표집한다.
  6. 별도로 narrative intensity vector와 missingness profile을 부여한다.
  7. 주변 분포가 아니라 조건부·결합 분포까지 audit한다.

작은 코호트의 자연 변동

확률 1% 사건은 100명에서 기대값이 1명이지만 0명일 확률도 약 36.6%다. observational 모드에서는 정확한 quota보다 binomial/Dirichlet-multinomial 기반 stochastic envelope를 사용한다. 반대로 coverage 모드에서는 최소 1건을 보장하되 실제 유병률로 오해되지 않도록 가중치를 기록한다.

12.2 임상 전방 시뮬레이터

latent_state(t0)
  → pathophysiologic transition
  → symptom manifestation
  → patient/caregiver perception
  → care-seeking decision
  → observation process
  → clinician knowledge update
  → intervention decision
  → treatment-modified state
  → new observation
  → disposition
  • Latent state: 생성기의 실제 상태. 문서 생성기에서 숨긴다.
  • Manifestation: 상태가 모든 증상을 완전하게 드러내지 않도록 확률적 관찰 가능성을 적용한다.
  • Observation process: 질문·측정 여부, 측정 성공, 장비 오차, 기록 여부를 별도 생성한다.
  • Treatment confounding: 수액·산소·해열제 이후 관찰은 처치 전 상태를 대표하지 않는다.
  • Knowledge update: 새로운 evidence가 들어올 때 hypothesis rank가 바뀐다.

12.3 시간 그래프 알고리즘

  1. 모든 event를 TemporalExtent와 clock source로 정규화한다.
  2. 확정된 before edge로 DAG를 구성한다.
  3. Kahn topological sort로 cycle을 검사한다.
  4. 구간 시간이 겹치는 경우 overlaps로 보존하고 임의 순서를 만들지 않는다.
  5. 문서 time과 event effective time, recorded time을 분리한다.
  6. 장비·서버 시계 offset이 알려지면 정규화값과 원시값을 함께 보존한다.

12.4 Knowledge frontier

구급일지에는 병원 도착 후 확정된 검사·진단이 들어갈 수 없다. 초진기록지에는 이후 입원 경과나 최종 진단이 들어갈 수 없다. 단, “나중에 추가 기재한 addendum”은 별도 document version과 recorded time으로 표현한다.

12.5 Claim consolidation

  1. 동일 subject–predicate–time window의 Claim 후보를 검색한다.
  2. 출처 독립성, measurement quality, 시간 근접성, 직접성으로 evidence weight를 계산한다.
  3. 합의 가능한 값은 consolidated Claim을 만들되 원본 Claim을 유지한다.
  4. 임계 차이를 넘는 값은 conflicts_with와 Unknown을 생성한다.
  5. 후속 evidence가 들어오면 새 consolidated Claim이 이전 Claim을 supersede한다.

12.6 Targeted regeneration

분포를 맞추기 위해 leaf 값을 직접 바꾸지 않는다. 변경할 root variable과 그 downstream dependency closure를 계산한다.

repair_loss =
    w1 * marginal_distance
  + w2 * joint_distance
  + w3 * temporal_distance
  + w4 * missingness_distance
  + w5 * duplication_penalty
  + λ  * regeneration_cost

candidate repair:
  root = blueprint.acuity
  closure = {
    clinical_state, observations, decisions,
    interventions, disposition, document_projections

후보 환자별 예상 개선량과 재생성 비용을 비교해 최소 집합을 선택한다. 구현은 초기에는 greedy selection으로 시작하고, 복잡한 quota 조합이 필요할 때만 min-cost flow 또는 mixed-integer optimization을 도입한다.

운영 가능성 판정과 단계적 생성 정책

이 설계는 교육·시뮬레이션용 합성 데이터 서비스로는 운영 가능하다. 실제 환자 진료를 자동 결정하거나 법정 원본 기록을 무인 대체하는 제품으로는 승인되지 않는다. 운영 전제는 합성/실데이터 환경 분리, 모든 산출물의 synthetic=true 표식과 watermark, 임상 검토자 승인 경계, 모델·카탈로그·프롬프트 버전 고정, 개인정보·라이선스·보안 점검이다.

대량 생성은 2-pass를 기본값으로 한다. Pass 1은 결정론적 Blueprint와 Chronicle 골격을 생성하고 코호트 분포·조건부 연관·결측률·극적 강도·중복 motif를 감사한다. Pass 2는 부족한 strata만 선택적으로 재생성한 뒤 승인된 Claim을 LLM으로 문장화한다. 이 방식은 100명 모두가 극적인 사례가 되는 mode collapse를 줄이면서 개별 기록의 풍부함을 유지한다.

운영 gate: 분포 감사 실패 시 문서화를 진행하지 않고 Blueprint 단계로 되돌린다. LLM 장애 시 템플릿 기반 구조화 출력으로 degrade하며 임상 사실을 새로 추론하지 않는다.

핵심 clinical invariants

  • 환자 접촉·측정·처치는 해당 encounter 범위 안에 있어야 한다.
  • 처치 반응 Observation은 처치 시작 이후여야 한다.
  • SBP와 DBP는 동일 measurement context에서 생성되며 일반적으로 DBP가 SBP를 초과하면 critical finding이다.
  • 입원 시각은 ED 도착 이후이며 입원 disposition과 함께 존재해야 한다.
  • 귀가 encounter에 입원 후 결과를 생성하지 않는다.
  • 검사 결과는 order 없이도 외부검사로 존재할 수 있으나 source encounter가 필요하다.
  • 확정 진단은 충분한 evidence 또는 명시적 clinical assertion을 요구한다.
  • 음성 소견은 질문·관찰 수행 근거가 있어야 한다.
  • NEDIS 조건부 필드는 질병/손상, disposition, arrival mode 조건을 따른다.

Privacy release gate

  • 공개 통계·규칙만 사용하는 fully synthetic 모드와 실제 record-level data를 학습하는 모드를 분리한다.
  • 실데이터 기반 패턴 학습 환경은 합성 결과도 release 전까지 개인정보 취급 수준으로 보호한다.
  • 직접식별자는 생성 단계부터 합성하고 실제 이름·전화·주소·면허번호를 prompt에 넣지 않는다.
  • 희귀한 속성 조합과 시간·지역 topology가 재식별 신호가 될 수 있으므로 direct identifier 제거만으로 충분하지 않다.
  • 실제 source row와의 nearest-neighbor distance, exact/near duplicate, membership inference 위험을 평가한다.
  • 외부 배포본은 목적에 맞는 필드 최소화, 일반화, suppression, sampling 정책을 적용한다.
  • 법적 적합성은 제품 기능이 아니라 별도 governance 승인 사항이다. 이 문서는 법률 자문을 대체하지 않는다.

Human review

  • 임상의 blind review: 빈도 기반 층화 표본과 극단·충돌 표본을 함께 평가.
  • 문서 실무자 review: 초진·구급·NEDIS 양식 적합성.
  • 데이터 review: target source, association, 결측, code mapping.
  • LLM judge는 보조 finding만 생성하며 단독 승인자가 될 수 없다.

  • Property-based: 임의 sequence에서도 시간 DAG, cardinality, supersede invariant 유지.
  • Contract: JSON Schema, API, document profile, NEDIS tables.
  • Golden patient: 임상 검토된 30–100개 Chronicle fixture.
  • Metamorphic: 나이·성별·기관·시간대를 바꿨을 때 의도된 관계만 변하는지 확인.
  • Cohort regression: 고정 seed 100·1,000·10,000명 분포와 metric snapshot.
  • Mutation: 미래 정보, 시간 역전, sentinel 혼입, Claim 충돌을 주입해 validator 탐지.
  • Human evaluation: 임상의·문서 실무자 blind review.

Release exit proof

  • 모든 schema migration이 빈 DB와 upgrade DB에서 성공한다.
  • critical invariant 및 knowledge leakage test가 100% 통과한다.
  • 문서 field lineage coverage가 필수 필드 100%, 전체 필드 목표치 이상이다.
  • 고정 benchmark cohort가 버전별 허용 tolerance를 통과한다.
  • 의도적으로 주입한 결함 fixture를 validator가 탐지한다.
  • 선택 재생성이 비영향 환자의 output hash를 바꾸지 않는다.
  • 동일 manifest replay가 canonical structured output을 재현한다.
  • privacy·licensing·clinical governance sign-off가 artifact로 남는다.
  • 운영 runbook, rollback, failed job recovery가 rehearsal을 통과한다.
